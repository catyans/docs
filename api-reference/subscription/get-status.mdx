# Get Subscription / Redemption Status

Track the progress of a subscription or redemption request through its lifecycle stages.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token. Requires `investor` role.
</ParamField>

## Path Parameters

<ParamField path="id" type="string" required>
  Subscription ID (`sub_*`) or Redemption ID (`red_*`)
</ParamField>

## Request

```bash
curl https://api.yourdomain.com/v1/subscription/status/sub_20260309_a1b2c3 \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..."
```

## Status Values

### Subscription Statuses

| Status | Description |
| --- | --- |
| `pending_payment` | Awaiting USDT transfer to payment address |
| `payment_received` | Payment detected on-chain |
| `confirmed` | AML/KYC checks passed |
| `on_chain` | Share allocation recorded on-chain |
| `active` | Shares active in portfolio |
| `rejected` | Failed AML/KYC or payment timeout |

### Redemption Statuses

| Status | Description |
| --- | --- |
| `pending_review` | Awaiting PM review |
| `approved` | Approved, awaiting settlement date |
| `processing` | Settlement in progress |
| `settled` | USDT sent to destination wallet |
| `completed` | On-chain confirmation received |
| `rejected` | Rejected (lock period, liquidity, etc.) |
